Methodology

GoBFD benchmarks use go test -bench -benchmem -count=6 with benchstat analysis. All results are median of 6 iterations on an isolated container.

C benchmarks (FRR/BIRD) are standalone programs that replicate the exact code patterns (packet fill, FSM switch, jitter calculation) extracted from FRR bfdd and BIRD BFD source code. They are NOT the full FRR/BIRD programs — they isolate the hot-path operations for apples-to-apples comparison of code patterns.

Python benchmarks (aiobfd) replicate the bitstring.pack() and ConstBitStream.unpack() patterns from aiobfd. The "struct" variant uses struct.pack()/unpack() as an optimized Python baseline.

These benchmarks measure isolated operations, not end-to-end system performance. Real-world performance depends on OS scheduling, network stack, socket I/O, and many other factors.

Goroutine-per-Session Model

GoBFD runs each BFD session in an isolated goroutine with a dedicated event loop. Incoming packets are delivered via buffered channels. This provides complete session isolation: one session cannot block another, and BFD timers are not affected by CPU load from other protocols.

C implementations (FRR, BIRD) run all BFD sessions in a single thread with a shared event loop. While faster per-packet, this means a CPU-intensive operation (e.g., BGP convergence) can delay BFD timer processing, potentially causing false session drops (FRR Issue #9078).

FullRxPath: Different Operations

Go FullRxPath (~50ns) = unmarshal + RWMutex lock + map lookup + channel send to session goroutine. C FullRxPath (~5ns) = unmarshal + inline FSM call (no IPC, no locking). The ~10x gap is the cost of goroutine isolation, not codec inefficiency.

FullRxPathCodec (Go-only) measures only unmarshal + FSM — the fair comparison with C. Expected ratio: ~1.5-2x, which is typical for Go vs C on compute-bound operations.

SessionDemux: Channel Send vs Hashmap Lookup

Go Demux (~47ns) = RWMutex + map lookup + channel send. C Demux (~1.4ns) = hashmap lookup only. ~70% of Go's cost is the channel send — which is the mechanism that enables goroutine isolation.

SessionLookup1000 (Go-only) measures only RWMutex + map lookup — the fair comparison with C's hashmap lookup.

SessionCreate: Different Abstractions

Go creates goroutines, channels, contexts, and loggers per session. C allocates structs and inserts into a hashmap. These measure fundamentally different operations and should not be compared directly.
