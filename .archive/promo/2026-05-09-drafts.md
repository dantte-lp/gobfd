## 1. osrg/gobgp#2563 — Native BFD support

> **Tone read**: fujita's comments are short and direct. He asks
> structural questions ("why peer-scoped over interface-scoped"),
> rarely thanks people effusively. Keep it factual.

```
For people landing on this thread now: the "separate BFD daemon over
gRPC" path that fujita and dylandreimerink discussed upthread is what
GoBFD does today.

https://github.com/dantte-lp/gobfd

`v0.6.0` ships RFC 5880/5881/5882/5883/7419/9384/9468/9747/9764, with
the GoBGP integration calling DisablePeer/EnablePeer on BFD state
changes (RFC 5882 §3.2 flap dampening). Runnable demo at
deployments/integrations/bgp-fast-failover/.

Echo (RFC 9747) and Micro-BFD (RFC 7130) live behind their own gRPC
services because their timer / aggregation models do not fit the base
AddSession shape.

#3388 takes the in-process route; both have their place. If a small
gRPC verb on GoBGP would help the standalone path (e.g. richer
state-change notifications during graceful restart), I can draft a PR.

Disclosure: I'm the author.
```

## 3. siderolabs/talos#11126 — Native GoBGP integration

> **Tone read**: smira asked for three specific deliverables (machine
> config change, draft PR, integration test approach). Match that
> structure when offering help.

```
maxpain — re: "lack of BFD as the main GoBGP gap" — there's a
standalone BFD daemon that closes that part of the gap right now:
https://github.com/dantte-lp/gobfd

`v0.6.0` runs as a sibling process to GoBGP and drives
DisablePeer/EnablePeer on BFD state changes. Sub-second BGP failover
becomes a function of BFD detection time rather than the BGP holdtimer.

For Talos packaging this looks like a natural extension candidate
under siderolabs/extensions, similar to bird2 (PR siderolabs/extensions#911).
The BFD daemon needs CAP_NET_RAW and an interface name; no kernel
modules. I can put up a draft extension manifest + boot test if that's
useful — it would parallel maxpain's GoBGP-as-extension story, with
GoBFD running next to it for liveness.

Disclosure: I'm the GoBFD author. Not pushing this as a replacement for
the native FRR/GoBGP work smira sketched in
https://github.com/siderolabs/talos/issues/11126#issuecomment-2929664808;
just naming a piece that already exists.
```

  on the issue threads above. Posting before community signal looks
  like astroturfing.

- **NANOG mailing list** — appropriate after at least one production
  deployment outside maintainer's own infra.
